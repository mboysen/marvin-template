## Major decisions / outcomes

- **Geometry is the binding constraint, not the model.** Every perception expert independently pointed at boundary-box coverage; mechatronics arrived at the same place from the velocity-provenance angle. Don't retrain. Don't tighten thresholds. Extend boxes + use correct velocity formula → expected 26% → 50%+.
- **Strict-PATCH is the right contract.** The endpoint-local fix in `config.py` ships today; the architectural generalization (delete `_default_config()`, single `ConfigService`, parameterize the PATCH idempotency contract over all 6 routes) is queued.
- **SW staleness is a structural class, not a per-release issue.** No-cache headers shipped today; generate-SW-from-`__version__` + bytecode parity test queued.
- **TRT is deferred.** `torch.compile(dtype=float32)` A/B happens first (hours of work) — gives a quantitative floor on what compile alone gets before pulling the TRT trigger (days of work).
- **GMSL camera swap accelerated to Q2.** UVC `frame_age_ms` p50 of ~119 ms is a structural ceiling; no software change reaches it. GMSL + zero-copy GPU mat takes that to ~20 ms — single biggest latency win available.
- **JET_LOG_DETECTIONS goes behind a debug flag.** Per-frame `fsync()` at 5 fps × 50+ detections/frame is eating the latency budget it was instrumenting. The 8.5 GB / disk-full incident today was the second-order effect.
- **`marks_team_of_experts` is now a reusable skill.** Pattern: 5 distinct PhD lenses, parallel background spawn in a single Agent batch, mandatory ranked-pick output format, synthesis is the orchestrator's job not a 6th agent's. Documented anti-patterns and lessons learned from this canonical run.

## Lessons learned

(Operational/process — separate from the technical Pareto picks at `ai-precise/docs/learnings/2026-05-27.md`.)

- **Browser-cache debugging eats hours.** The SW staleness class cost ~3 hours today and ~3 hours on each of the last 3 releases. The middleware fix is correct; the bytecode-parity test (item #4 in synthesis) is what makes the fix irreversible.
- **Strict-PATCH or strict-anything beats silent-fallback.** `_default_config()` silently merging onto an unreadable yaml blanked fields three times today. Refusing with 503 is louder, sooner, more debuggable.
- **Operator-drawn boundaries need visual feedback at the FOV level, not just the box level.** First yard test showed `100% GEOMETRY_BOTH_FAIL` because operator drew all 8 boxes in the bottom 20% and weeds appeared everywhere else. A FOV-coverage heatmap overlay would catch this before the test.
- **5 PhD subagents in parallel >> 1 generalist agent.** The 5-lens cross-cutting view surfaced bug classes (E-stop chain, JET_LOG_DETECTIONS fsync, `_default_config()` fallback) that a single review would have missed. Captured as the `marks_team_of_experts` skill.
- **Heredoc your shell commands when they contain `cd` over SSH.** The pattern `ssh host 'cd /path && cmd'` had the `cd` stripped repeatedly today. Switched to `ssh host bash <<'REMOTE' ... REMOTE` which works reliably.
- **Free disk before the rebuild.** Today's `:rollback` Docker tag was holding 23 GB; dropping it freed enough space to rebuild in-place without a tarball detour. Schedule a disk-pressure check into pre-rebuild.
- **Don't read agent JSONL transcripts directly while the agent runs.** Overflows the orchestrator's context. Wait for the task-notification, process the result block in the notification.
- **Synthesis is the orchestrator's job, not a 6th agent's.** Each lens gives a perspective; the cross-cutting view is what the user is paying for. A 6th agent would just be another lens.
